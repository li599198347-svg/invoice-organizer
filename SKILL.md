# 差旅发票整理技能

从 QQ 邮箱自动提取发票，生成报销用 Excel 和 Word 打印版。

## 核心功能

### ✅ 已实现并测试

| 发票类型 | 识别方式 | 附件处理 | 提取内容 | 状态 |
|---------|---------|---------|---------|------|
| **滴滴** | 发件人含 `xiaojukeji.com` | 发票 PDF + 行程单 PDF | 发票号、金额、行程日期、起点、终点 | ✅ |
| **阳光出行** | 主题含"第三方"/"阳光出行" | 发票 PDF + 行程单 PDF | 发票号、金额、行程日期、起点、终点 | ✅ |
| **通行费** | 发件人含 `txffp.com` | ZIP(多张子发票) + 行程单 PDF | 多张发票号、总金额、入口站、出口站 | ✅ |
| **携程机票** | 主题含"携程"或发件人含 `trip.com` | 发票 PDF | 发票号、金额（简化处理） | ✅ |
| **12306 火车票** | 主题含"火车票"/"高铁"/"铁路"或发件人含 `12306` | 发票 PDF | 发票号、金额、行程日期、出发站、到达站 | ✅ |

### 特殊场景处理

| 场景 | 处理方式 |
|------|---------|
| 一张发票对应多个行程 | Excel 显示"对应 X 个行程" |
| 发票 PDF 无行程信息 | 从同一封邮件的行程单 PDF 提取 |
| 通行费多张子发票 | Excel 汇总一行，Word 每张单独成页 |
| 多发票邮件（A/B） | 按文件名配对行程单 |

## 配置

### QQ 邮箱 IMAP 设置

1. 登录 QQ 邮箱 → 设置 → 账户 → POP3/IMAP/SMTP 服务
2. 开启 **IMAP/SMTP 服务**
3. 生成 **授权码**（不是邮箱密码）

### 写入记忆文件

在 `~/.openclaw/workspace/MEMORY.md` 中添加：

```markdown
## 邮件配置
IMAP_HOST=imap.qq.com
IMAP_PORT=993
IMAP_USER=你的 QQ 邮箱
IMAP_PASS=你的授权码
IMAP_MAILBOX=INBOX
```

## Word 打印版格式

- **内容**：`Page X - 发票号 (n/总数)`

## 处理逻辑详解

### 1. 邮件扫描与类型识别

```python
# 优先级：发件人域名 > 主题关键词
if 'xiaojukeji.com' in sender:
    inv_type = '阳光出行' if '第三方' in subject else '滴滴'
elif 'txffp.com' in sender:
    inv_type = '通行费'
elif '携程' in subject or 'trip.com' in sender:
    inv_type = '携程机票'
elif '火车票' in subject or '高铁' in subject or '12306' in sender:
    inv_type = '12306 火车票'
```

### 2. 行程单提取逻辑

```
行程单 PDF 结构（多行格式）：
├── 表头：序号 | 车型 | 上车时间 | 城市 | 起点 | 终点 | 里程 | 金额
└── 数据行：1 | 滴滴特快 | 03-15 14:49 | 深圳市 | 蛇口邮轮中心... | 宝安机场... | 25.3 | 69.90

行程单 PDF 结构（单行格式）：
└── 数据行：1 | 滴滴特快 03-13 16:22 周五 深圳市 | 起点 | 终点 | 8.6 | 21.50

提取步骤：
1. 找到"里程"数字行（如 25.3 或 8.6）
2. 向前找序号（如 1）
3. 序号和里程之间的内容：车型→时间→城市→起点→终点
4. 城市特征：常见城市名（北京/上海/广州/深圳等）或含"市"的短词
5. 城市之后的内容平分：前半=起点，后半=终点
6. 从"行程起止日期"提取实际行程日期（优先于邮件日期）
```

### 3. 多发票↔多行程单匹配

```
邮件 51 示例（一张发票对应多个行程）：
├── 滴滴电子发票 A.pdf (¥1682.98)
├── 滴滴电子发票 B.pdf (¥90.91)
├── 滴滴出行行程报销单 A.pdf (19 个行程)
└── 滴滴出行行程报销单 B.pdf (2 个行程)

匹配规则：
- 文件名含 A → 发票 A 对应 行程单 A → Excel 显示"对应 19 个行程"
- 文件名含 B → 发票 B 对应 行程单 B → Excel 显示"对应 2 个行程"
- 无字母后缀 → 所有行程归到第一张发票
```

### 4. 通行费 ZIP 处理

```
ZIP 内容：
├── xml/26317907100300133055.xml  # 发票数据
├── xml/26317907090300037055.xml
├── pdf/26317907100300133055.pdf  # 发票 PDF
└── pdf/26317907090300037055.pdf

提取逻辑：
1. 遍历所有 XML 文件
2. 提取 <EIid>发票号</EIid>
3. 提取 <TotaltaxIncludedAmount>金额</TotaltaxIncludedAmount>
4. 累加所有子发票金额
5. 保存所有子发票 PDF 到"发票 PDF"文件夹
6. Word 中每张子发票单独一页
```

### 5. 携程机票简化处理

```
携程机票处理策略：
- 只从文件名提取订单号：订单 1128146092441008-电子普通发票.pdf → 携程1128146092441008
- 只提取金额：从 PDF 中提取合计金额
- 不提取行程信息：HTML 格式复杂，显示"详见发票"
- 原因：节省处理时间，用户可手动补充行程
```

### 6. 12306 火车票处理

```
12306 火车票提取：
- 发票号：从 PDF 提取"发票号码"字段
- 行程日期：从 PDF 提取日期字段
- 出发站：从 PDF 提取"出发站"字段
- 到达站：从 PDF 提取"到达站"字段
- 金额：从 PDF 提取"票价"字段
```

## 常见问题

### Q: 行程单提取不到路线信息
**A:** 检查行程单 PDF 格式是否标准。支持两种格式：
- 多行格式：城市、起点、终点各占一行
- 单行格式：所有内容在一行，用空格分隔

### Q: 通行费 Word 中只显示第一张发票
**A:** 已修复。现在通行费 ZIP 中所有子发票都会在 Word 中单独成页。

### Q: 一封邮件多张发票只处理了一张
**A:** 已修复。现在支持文件名 A/B 后缀的配对逻辑。

### Q: 如何修改输出目录
**A:** 修改 `scripts/main.py` 中的 `OUTPUT_DIR` 变量：
```python
OUTPUT_DIR = os.path.join(os.path.expanduser('~/Desktop'), '差旅发票')
```

### Q: 携程机票为什么没有行程信息
**A:** 携程机票 HTML 格式复杂，为节省处理时间采用简化策略，只显示发票号和金额。如需行程信息，可手动补充。
